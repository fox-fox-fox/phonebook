Test - Phonebook

The test is the implementation of a simple phonebook composed of the following screens:

Home
Contact list
Add Contact
Edit Contact
Delete Contact

Comments:
  • A telephone contact contains at least: name, email and telephone
  • The list can be public (no login required);
  • Contact management functions (Add, Edit, Delete) do not necessarily have to be a page
  • Forms actions are expected to be sent via REST API
  • If you don't choose to send a form via REST API you can handle the data by localStorage
  • Mockups can be used to request data
Objective:

The intent of the test is to assess your analysis skills, this is an excellent opportunity to show us your algorithm knowledge and problem-solving skills. In addition, you will be able to show off your skills with pure javascript, the most semantic HTML possible, and well-organized CSS.

Delivery:

The test can be sent as a compressed file in this thread or the project link in some versioning tool (github, bitbucket);
